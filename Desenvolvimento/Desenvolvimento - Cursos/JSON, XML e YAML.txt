XML 
	Objetivo foi a criação de um tipo de formato que poderia ser lido por softwares e que tivesse flexibilidade e simplicidade
	Possibilitou a criação de "tags" (você mesmo)
	concentração da informação na estrutura e não na aparência
	é um concorrente nessa troca de informações
	
	trabalha com tags, para delimitar os dados
	
	- Tanto XML quando JSON são modelos para representar dados no formato de texto;
	- Ambos possuem natureza auto-descrita (basta "bater o olho" em um arquivos desses tipos para entender seu significado);
	- Ambos são capazes de representar informações complexas no formato tubular. Alguns exemplos, objetos compostos (objetos dentro de objetos), relações de hierarquia , arrays, dados ausentes etc;
	- Ambos podem ser utilizados para transportar informações em aplicações AJAX;
	- Ambos podem ser considerados padrões para representação de dados. XML é um padrão W3C e o JSON é formalizado na RFC 4627.
	- Ambos são independentes de linguagem de programação
	- Podem ser acessados por qualquer linguagem de programação (algumas ainda não tem suporte a JSON) através de apis especificas
